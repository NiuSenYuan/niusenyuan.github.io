使用 GitHub Pages 搭建个人博客

📅 2026-04-28 ✍️ 博主 ⏱ 5 分钟阅读
GitHub 教程 博客搭建

GitHub Pages 是 GitHub 提供的免费静态网站托管服务。你可以用它来托管个人博客、项目文档、作品展示等。最重要的是——它完全免费，而且与 Git 工作流完美集成。

为什么选择 GitHub Pages？

  • 免费：无需购买服务器或域名（可以使用 username.github.io 子域名）
  • 简单：只需推送 HTML/CSS/JS 文件到 GitHub 仓库，网站即可自动部署
  • 版本控制：所有内容通过 Git 管理，方便回溯和协作
  • HTTPS：自动提供 SSL 证书，安全可靠

快速开始

第一步：创建仓库

在 GitHub 上创建一个新的仓库。如果你想要用户页面（个人主页），仓库名必须为 username.github.io（将 username 替换为你的 GitHub 用户名）。

如果你想要项目页面，可以创建名为 gh-pages 的分支，或者使用任意仓库名并在设置中开启 Pages 功能。

第二步：推送代码

将你的静态网站文件推送到仓库中：

git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/username/username.github.io.git
git push -u origin main

第三步：开启 GitHub Pages

进入仓库的 Settings → Pages，在 "Source" 中选择部署分支（通常是 main），点击保存。几秒钟后你的网站就会在 https://username.github.io 上线。

提示：你也可以绑定自己的自定义域名。在 Settings → Pages 中设置 Custom domain，然后在 DNS 服务商处添加 CNAME 记录即可。

博客结构建议

一个基本的博客文件结构：

blog/
├── index.html          # 首页
├── posts.json          # 文章列表数据
├── css/
│   └── style.css       # 样式
├── js/
│   └── main.js         # 脚本
├── posts/              # 文章目录
│   ├── post-1.html
│   └── post-2.html
└── images/             # 图片资源

总结

GitHub Pages 是一个非常棒的静态网站托管方案，特别适合技术人员。你现在看到的这个博客，就是完全运行在 GitHub Pages 上的。只需要写好 HTML 和 CSS，推送到 GitHub，一切就搞定了。
