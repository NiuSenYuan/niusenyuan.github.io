Python 异步编程入门：从回调到 async/await

📅 2026-04-25 ✍️ 博主 ⏱ 8 分钟阅读
Python 异步 编程

异步编程是现代 Python 开发中不可或缺的技能。无论是 Web 服务器、网络爬虫还是数据处理，合理使用异步编程都能显著提升性能。

为什么需要异步？

在传统的同步编程中，程序会按顺序执行每一行代码。当一个操作需要等待 I/O（如网络请求、文件读写）时，整个程序会被阻塞，浪费大量的 CPU 时间。

异步编程允许程序在等待 I/O 操作时切换到其他任务，从而充分利用 CPU 资源。

基础概念

协程 (Coroutine)

协程是使用 async def 定义的函数。调用协程函数会返回一个协程对象，需要用 await 或事件循环来执行它。

import asyncio

async def greet(name):
    await asyncio.sleep(1)
    return f"Hello, {name}!"

async def main():
    result = await greet("World")
    print(result)

asyncio.run(main())  # 输出: Hello, World!

并发执行多个协程

使用 asyncio.gather() 可以并发运行多个协程：

import asyncio
import time

async def fetch_data(id, delay):
    await asyncio.sleep(delay)
    return f"Data from {id}"

async def main():
    start = time.time()
    results = await asyncio.gather(
        fetch_data("A", 2),
        fetch_data("B", 1),
        fetch_data("C", 3),
    )
    elapsed = time.time() - start
    print(results)
    print(f"耗时: {elapsed:.1f}s")  # 约 3s，而非 6s

asyncio.run(main())

实战：异步 HTTP 请求

使用 aiohttp 库进行异步 HTTP 请求：

import asyncio
import aiohttp

async def fetch(url):
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as response:
            return await response.text()

async def main():
    urls = [
        "https://httpbin.org/delay/2",
        "https://httpbin.org/delay/1",
    ]
    results = await asyncio.gather(*[fetch(u) for u in urls])
    print(f"获取了 {len(results)} 个页面")

asyncio.run(main())

常见陷阱

  • 不要在协程中使用阻塞操作：如 time.sleep() 会阻塞事件循环，应使用 await asyncio.sleep()
  • 注意异常处理：使用 try/except 包裹 await 表达式
  • 合理设置并发数：使用 asyncio.Semaphore 限制并发数量

总结

Python 的 async/await 语法让异步编程变得简单直观。掌握它，你就能写出高效、响应迅速的 Python 程序。
